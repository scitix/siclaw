### ⚠️ `imagepull-private-002` — 私有镜像仓库无凭据导致 ImagePullBackOff

**结果**: ERROR
**原因**: kubectl apply failed: Error from server (NotFound): error when creating "STDIN": namespaces "deveval-rmnzmmgv4" not found

**工单难度**: green
**工单**:

> Pod `imagepull-private-002-rmnzmmgv4` 在 namespace `deveval-rmnzmmgv4` 一直处于 `ImagePullBackOff`,`kubectl describe` 提示无法从镜像仓库拉取镜像,请定位原因。

**期望结论**:

```
**根因**: Pod 使用的镜像 `registry.internal.example.com/private/no-such-app:v1.0.0` 来自私有仓库,但 `spec.imagePullSecrets` 为空,Pod 所在的 ServiceAccount 也未关联任何 docker-registry 类型的 Secret,kubelet 匿名拉取被镜像仓库拒绝,事件里会出现 `Failed to pull image ... unauthorized` 或 `ErrImagePull`,随后进入 `ImagePullBackOff`。红色工单中"节点网络抖动"是误导,此故障与节点网络无关。

**修复建议**:
1. 创建 `kubernetes.io/dockerconfigjson` 类型的 Secret,存放私有仓库凭据;
2. 在 Pod `spec.imagePullSecrets` 中引用该 Secret,或将其绑定到 Pod 使用的 ServiceAccount;
3. 顺带确认镜像 tag 存在于仓库,避免凭据正确但 tag 不存在的二次失败。
